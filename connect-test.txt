================================================================
  Cursor AI + Claude Code + MCP 완전 가이드
================================================================

----------------------------------------------------------------
1. Cursor AI에서 Claude Code(MCP) 연동하는 법
----------------------------------------------------------------

[1단계] MCP 서버 설치 (터미널에서 실행)
  npm install -g @anthropic-ai/claude-code

[2단계] Cursor 설정에서 MCP 추가
  Cursor > Settings > MCP (또는 .cursor/mcp.json 파일 생성)

  .cursor/mcp.json 예시:
  {
    "mcpServers": {
      "claude-code": {
        "command": "claude-code",
        "args": ["mcp-server"]
      }
    }
  }

[3단계] Cursor 재시작 후 MCP 활성화 확인
  - Cursor 하단 상태바에 MCP 아이콘 확인
  - 채팅창에서 @claude-code 로 호출 가능

----------------------------------------------------------------
2. 일반 채팅 vs MCP: 무엇이 달라지나?
----------------------------------------------------------------

  일반 채팅 (지금 이 방식)     │  MCP 연동
  ─────────────────────────────┼──────────────────────────────
  파일 읽기/쓰기 (허가 필요)   │  자동으로 파일 시스템 접근
  한 번에 하나의 작업           │  여러 작업을 연속으로 자동화
  컨텍스트 제한 있음            │  프로젝트 전체 컨텍스트 유지
  수동으로 파일 경로 제공       │  프로젝트 구조 자동 인식
  코드 실행 불가                │  터미널 명령 직접 실행 가능
  매번 허가 요청                │  워크플로우 자동화 가능

  핵심 차이: MCP는 Claude가 "도구(Tool)"를 사용할 수 있게 됨
  → 파일 시스템, 웹 검색, DB, API 등을 직접 호출

----------------------------------------------------------------
3. MCP 기본 구조
----------------------------------------------------------------

  [Client]  ←→  [MCP Server]  ←→  [Tools/Resources]
   Cursor          Claude              파일시스템
   VSCode          Code                웹 브라우저
   앱 등                               DB, API 등

  MCP Server가 제공하는 3가지 요소:
  ┌─────────────┬────────────────────────────────────────┐
  │ Tools       │ Claude가 호출할 수 있는 함수           │
  │             │ 예: read_file, run_command, search_web │
  ├─────────────┼────────────────────────────────────────┤
  │ Resources   │ Claude가 읽을 수 있는 데이터            │
  │             │ 예: 파일 내용, DB 레코드, API 응답      │
  ├─────────────┼────────────────────────────────────────┤
  │ Prompts     │ 재사용 가능한 프롬프트 템플릿           │
  │             │ 예: 코드 리뷰 양식, 커밋 메시지 형식    │
  └─────────────┴────────────────────────────────────────┘

----------------------------------------------------------------
4. Agent 구조 (Claude Agent SDK)
----------------------------------------------------------------

  단순 채팅:  User → Claude → 답변

  Agent:      User → Orchestrator Agent
                         ↓
                  [Sub-Agent 1]  [Sub-Agent 2]  [Sub-Agent 3]
                  파일 분석       웹 검색         코드 실행
                         ↓
                  결과 취합 → User

  Agent 유형:
  ┌──────────────────┬───────────────────────────────────┐
  │ Orchestrator     │ 전체 작업을 계획하고 분배          │
  │ Subagent         │ 특정 작업 전문 처리                │
  │ Tool-use Agent   │ MCP 도구를 직접 사용               │
  │ RAG Agent        │ 문서/DB에서 정보 검색              │
  └──────────────────┴───────────────────────────────────┘

----------------------------------------------------------------
5. 비용 효율적으로 사용하는 법
----------------------------------------------------------------

  [토큰 절약 전략]

  ① 모델 선택 전략 (비용 차이 큼!)
     - 간단한 작업  → claude-haiku-4-5  (가장 저렴)
     - 일반 개발    → claude-sonnet-4-6 (균형)
     - 복잡한 설계  → claude-opus-4-6   (가장 비쌈)

  ② 컨텍스트 최소화
     - 필요한 파일만 @멘션
     - 긴 파일은 필요한 부분만 잘라서 전달
     - 대화가 길어지면 새 채팅 시작

  ③ 캐싱 활용 (Prompt Caching)
     - 동일한 시스템 프롬프트 반복 사용 시 90% 할인
     - Claude API에서 cache_control 설정

  ④ MCP로 자동화 = 비용 절감
     - 반복 작업을 Agent에게 위임
     - 한 번의 요청으로 여러 파일 동시 처리
     - 수동으로 복붙하는 시간/비용 제거

  ⑤ 배치 처리 (Batch API)
     - 급하지 않은 작업은 Batch API 사용 → 50% 할인
     - 예: 대량 코드 리뷰, 문서 생성 등

  [실전 팁]
  - /compact 명령으로 대화 압축 (컨텍스트 절약)
  - CLAUDE.md 파일로 반복 설명 없애기
  - 작은 작업은 Haiku, 큰 작업만 Opus/Sonnet 사용
